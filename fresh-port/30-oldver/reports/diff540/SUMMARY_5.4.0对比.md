# 5.4.0 ↔ 5.5.0 对比总览(夏季海洋更新 = 最后一次版本跳变)

综合 3 份分节(同目录 `diff540_code.md` / `diff540_assets.md` / `diff540_data.md`)。5.4.0:armv7/脱壳/2178文件/1963类/36250法;5.5.0:2188文件(原始)/1968类/36127法。**纯调研,未改任何代码/游戏资源。**

## 一句话结论
**5.4.0→5.5.0 是一次纯内容补丁:加了一整套"海底寻宝"联网活动 + 35 个海洋主题物品,零删除、零重构、核心逻辑一字未动。5.5.0 是 5.4.0 的严格超集 = 正确的移植目标;5.4.0 没有任何切换价值。**

## 代码(diff540_code)
- 末版**只新增 5 个类**,全是海底寻宝:`SeabedSeekingTreasureData/MainLayer/ExchageRewardLayer/RuleLayer` + `GetItemRewardFromHaiwangLayer`。5.4.0 的 1963 类在 5.5.0 全保留(零删除)。
- `GameData` +20 法(海底寻宝状态:珍珠数/已挖贝壳/开关/标志位)、`NetworkManager` +14 法(海底寻宝联网协议)。其余"方法增删"是 category 宿主归属伪影,剔除后真实删除=0。
- 海底寻宝是**纯联网玩法**(珍珠坐标/挖贝壳/兑换/领奖全走服务器)→ 离线需打桩或屏蔽入口。
- **无 Caribbean/黄金岛新类**(海洋更新没补)。

## ★等级/数值混淆链:5.4.0 与 5.5.0 逐字节相同(对修等级 bug 无价值)
- 对 12 个关键函数(`encryptInt:`/`decryptInt:`/`encrypt:key:options:`/`addXp:`/`checkUpgrade`/`curLevel`/`encryptCurLevel`/`setNewCurLevel:`/`setCurLevel:`/`encryptVipGold`/`WrapperManager addXp:`)逐函数 disasm 对比:**指令骨架 diff 全 0、立即数 diff 全 0**。
- 核心 XOR 原语 `encryptInt:`/`decryptInt:` 两版机器码**逐字节完全相同** = `f2410011 f2c01001 4050 4770`,key `0x01011011`,自反(encrypt=decrypt)。访问器仅有的 2~4 字节微差 100% 是 `movw/movt` 字面量池偏移(因 5.5.0 多 5 类布局位移),语义零变化。
- **判定**:数值混淆是 2.4.3→5.4.0 之间加的;5.4.0 和 5.5.0 是同一份等级链代码。"等级不涨"在 5.4.0 同样存在,**非末版引入,5.4.0 对修此 bug 无额外参照**。修复仍锁定 **touchHLE 运行时层的 XOR 解码 / checkUpgrade**,而非换版本。

## 对项目的结论
1. **继续用 5.5.0 移植**——它是 5.4.0 超集,换 5.4.0 只会丢海洋内容(35物品+水上乐园)、一无所得。
2. **等级不涨**:5.4.0 实锤了根因不在"版本代码差异"(5.4.0=5.5.0),而在 touchHLE 对 XOR 混淆/checkUpgrade 的运行时处理——修复要在模拟器侧。已知 XOR 原语机器码 `f2410011 f2c01001 4050 4770` / key `0x01011011`。
3. **无新资源可补**(海洋更新自洽);黄金岛仍无离线解(加勒比数据在但活动美术服务器独有,5.4.0 同样)。
4. 可选清理:5.5.0 包里历史遗留的 `hs_err_pid*.log`(JVM崩溃日志)、`*.decoded.plist`(解档产物)——非游戏原生,建议清掉。
