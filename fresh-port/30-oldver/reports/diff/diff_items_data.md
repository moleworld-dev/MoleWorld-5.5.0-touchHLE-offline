# 摩尔庄园 三版数据表对比报告(物品 / 建筑 / 任务 / 其他数据表)

对比版本:**1.1.5**(`Mole's World.app`) → **2.4.3**(`MoleWorld.app`) → **5.5.0**(`01-cracked/MoleWorld.app`)

解密方式:所有 `.dat` = AES-128-ECB,key = ASCII `39653543fa0d66aa`(hex `33393635333534336661306436366161`),解出为 Apple bplist。解密产物只写在 `/tmp/molediff/`,**未修改任何游戏文件**。

> 术语:`property.dat` 是**主物品/建筑表**;`propertyHV.dat` 是另一张**独立的 HV(HighVersion / 高级庄园)表**,ID 部分与主表重叠但不是主表的子集。`*HV.dat` 系列都是"高级庄园 / iPad-HD 玩法分支"的独立数据。

- **疑似弃用**:数据治理很干净——5.5.0 主表 1063 个 ID 里**只有 13 个无中文名**(且都不是可售商品,主要是 type=38 的 VIP 等级标记位),**只有 2 个物品**在 5.5.0 失去了售卖渠道(卖切糕的小车、雪球)。没有发现"数据里存在但被标隐藏可售"的隐藏商品。
- **某版独有数据表**:5.5.0 独有一大批 `2xx/3xx` 编号表(头像、活跃度、加勒比、合成、兑换、内购等)和 `vipquest`;1.1.5 独有 `xiaotulv_*`(丝尔特/小兔率岛屿玩法的预置存档)。

## 五、1.1.5 独有的明文/预置数据:`xiaotulv_*`(丝尔特 / 小兔率玩法)

`xiaotulv` = 拼音"小兔率",对应游戏内**丝尔特独立岛屿/小屋玩法**。两个文件都是 **NSKeyedArchiver 序列化的预置存档**(不是配置表):

- **`xiaotulv_map.dat`**:**明文 bplist**(无需解密),一个超大 NSDictionary 序列化(约 6800 个对象节点),是该玩法的**预置地图/场景布局存档**。
- **`xiaotulv_userinfo.dat`**:**AES 加密**(同一 key),解出为 NSKeyedArchiver 用户存档,含 `curLevel=37 / gold=252548 / availableWorkers=7 / curQuestId=15 / achieveUnlock` 等字段 —— 是一份**预置的丝尔特玩家档**(演示/初始存档)。

> 含义:1.1.5 把"丝尔特玩法"做成了一个**带预置存档的独立离线小世界**(自带一张地图 + 一个示例存档)。该玩法在 2.4.3 / 5.5.0 中**整个被移除**(后两版均无 `xiaotulv_*` 文件)。
